AI는 모델로 시작되지만, 현실에서는 시스템으로 검증됩니다.

연산 자원, 메모리, 데이터 품질, 운영 제약을 분리하지 않고 하나의 실행 구조로 통합하는 일을 해왔습니다. 목표는 언제나 동일합니다. 데모를 넘어 현장에서 반복 가능하게 동작하는 AI 시스템을 구축합니다.

AI 응용 시스템 엔지니어

01

핵심 관점

연산 제약

임베디드 환경에서 연산 자원과 메모리 제약은 옵션이 아니라 설계의 출발점입니다.

데이터 현실

산업 데이터는 대부분 AI 친화적 형태로 존재하지 않습니다. 정제와 흐름 설계가 우선입니다.

실행 구조

모델 정확도보다 중요한 것은 반복 실행성입니다. 동작을 보장하는 시스템 구조가 성능을 만듭니다.

AI는 정확도 이전에, 실행 가능해야 합니다.

02

실행 중심의 여정

  1. 단계 1

    하드웨어 / RISC-V / FPGA

    실행 환경의 물리적 제약을 체득

    초기 RISC-V 아키텍처와 FPGA에서 Linux 실행 검증을 수행하며 부트로더, 드라이버, 디버깅까지 저수준 계층을 직접 다뤘습니다.

  2. 단계 2

    NPU IP / SDK / 상용화

    모델을 제품으로 전환하는 체계 구축

    NPU IP 도입과 검증, SDK 설계 및 상용화를 통해 양자화, 실행 최적화, 인터페이스 설계를 반복 가능한 방식으로 정리했습니다.

  3. 단계 3

    자율주행 SoC / 기술 총괄

    다중 기관과 기술 계층의 정렬

    다중 센서 기반 자율주행 SoC 프로젝트에서 기술 총괄 역할로 NPU 실행 환경을 포함한 시스템 구조, 계층 간 책임, 데이터 흐름을 정합성 있게 설계했습니다.

  4. 단계 4

    산업 데이터 아키텍처

    현장 데이터 구조를 AI 실행 구조로 변환

    PLC, CNC, 로봇 환경에서 수집된 비정형 데이터를 시계열 아키텍처로 재구성하며 운영 제약을 반영한 AI 적용 기반을 마련했습니다.

03

실행 원칙

실행 우선

정확도 향상 이전에 배포 가능한 실행 경로를 먼저 설계합니다.

제약 기반 설계

연산/메모리/지연/운영 제약을 요구사항으로 격상시켜 아키텍처를 결정합니다.

시스템 정합성

하드웨어, 모델, 데이터, 운영 책임이 충돌하지 않도록 계층 간 인터페이스를 표준화합니다.

운영 반복성

실험 성공보다 현장 반복성을 우선해 프로세스와 모니터링을 설계합니다.

04

시스템 관점

하드웨어, 모델, 데이터, 운영은 분리된 블록이 아니라 상호 제약을 주고받는 실행 시스템입니다.

하드웨어 계층
